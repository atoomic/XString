# XString — Developer Guide

XS module providing `cstring()` and `perlstring()` from `B` as a standalone package.

## Build & Test

```bash
perl Makefile.PL && make && make test
```

Requires a C compiler. No non-core runtime dependencies.

## Architecture

- **`XString.xs`** — Single C function `cstring(pTHX_ SV *sv, bool perlstyle)` with two code paths:
  - UTF-8 + perlstyle: uses `sv_uni_display()` then processes the display string
  - Byte-by-byte: handles escapes, trigraphs (cstring only), and octal fallback
- **`lib/XString.pm`** — XSLoader bootstrap, POD documentation
- **`ppport.h`** — Perl portability header (Devel::PPPort)
- **`Makefile.PL`** — Generated by Dist::Zilla, committed for CI use

## XS Binding

`cstring` and `perlstring` are the same C function via ALIAS:
```
void cstring(sv)
    ALIAS:
        perlstring = 1
```
`ix` (alias index) controls the `perlstyle` boolean.

## Key Behaviors

- **undef** → returns `"0"` (matches `B::cstring`/`B::perlstring`)
- **Trigraphs** (`??X`): escaped in `cstring()` only (C trigraph prevention), not in `perlstring()`
- **`\v`**: escaped as `\v` in `cstring()` only; falls through to octal in `perlstring()`
- **`$` and `@`**: escaped in `perlstring()` only (Perl interpolation prevention)
- **UTF-8 flag**: `perlstring()` uses `sv_uni_display` for `\x{HH}` escapes; `cstring()` always takes the byte path

## Tests

- `t/00-load.t` — Module loading
- `t/cstring.t` — cstring comparison against `B::cstring` (0..255, UTF-8 upgraded, multi-char)
- `t/perlstring.t` — perlstring round-trip + `B::perlstring` comparison + Latin-1 vs UTF-8 path verification
- `t/trigraph.t` — All 9 C trigraph sequences + edge cases
- `t/threads.t` — Thread-safety verification (skipped if Perl lacks ithreads)

## Conventions

- Tests compare XString output against `B` module output — B is the reference implementation
- `is()` calls: got (XString) first, expected (B) second, always include description
- Pragmas: `use warnings; use strict;` — shebang `#!./perl` for test files
- Dist::Zilla for releases; `dist.ini` uses `[Git::NextVersion]` (versions from git tags: `v0.001`, etc.)
- MIN_PERL_VERSION is 5.008
- CI: GitHub Actions (`.github/workflows/ci.yml`) — dynamic Perl version matrix via `perl-actions/perl-versions@v2`

## Upstream sync (B.xs)

`XString.xs` is a verbatim port of the `cstring()` / `cchar()` C functions from
core Perl's `ext/B/B.xs`. B is the source of truth — when B fixes a bug or
adds a case, XString should follow.

Two mechanisms keep us honest:

1. **Test-time guard.** Every test in `t/` compares XString output against the
   locally installed `B::cstring` / `B::perlstring` / `B::cchar`. Any drift
   between XString and the user's Perl is caught here.
2. **Source-level audit.** Periodically (on Perl major releases, or when a
   B-related issue is reported), diff XString.xs against current blead:

   ```bash
   curl -sL https://raw.githubusercontent.com/Perl/perl5/blead/ext/B/B.xs \
     -o /tmp/upstream-B.xs
   # cstring/cchar live around lines 250-364 in upstream B.xs;
   # extract and compare against XString.xs (whitespace-only diffs are expected).
   diff <(sed -n '249,364p' /tmp/upstream-B.xs) <(sed -n '20,101p' XString.xs)
   ```

   Last verified in sync: **2026-04-16** (Perl/perl5 blead, modulo whitespace).

If a real divergence appears, port the upstream change to `XString.xs` and
update this date.
